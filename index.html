Frequently Asked Questions

When will MANEUVER be ready to use?

We're currently in the foundation phase. A basic interpreter should be available in 3-6 months for simple robots. Production-ready compiler expected in 18-24 months.

What platforms will MANEUVER support?

MANEUVER will compile to native code for Arduino, Raspberry Pi, NVIDIA Jetson, x86/ARM Linux, and more. Same code runs everywhere from microcontrollers to cloud servers.

Is MANEUVER really easier than Python?

For robotics tasks, yes! While Python is simple, it lacks robotics-specific features. MANEUVER reads like English but understands coordinates, units, and timing‚Äîmaking robot code much simpler.

How does performance compare to C++?

MANEUVER compiles to optimized native code via LLVM, achieving performance equivalent to well-written C++. Zero-cost abstractions mean high-level code doesn't sacrifice speed.

Can I use existing C++ libraries?

Yes! MANEUVER will have interoperability with C/C++ libraries. You'll be able to wrap existing robotics libraries (OpenCV, ROS, etc.) and call them from MANEUVER.

Is this just another academic project?

No. MANEUVER is designed for real-world robotics production. We're building for industry adoption with practical tooling, documentation, and community support.

How can I help if I'm not a compiler expert?

We need robotics domain experts, technical writers, designers, testers, and community builders just as much as compiler engineers. Everyone can contribute!
